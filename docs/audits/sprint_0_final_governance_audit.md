## Allocation Leakage Findings

Decision Engine contains allocation semantics by design:

- `final_action`
- `tradeability`
- `conviction`
- `allocation_priority`
- BUY/SELL/HOLD/TRIM/REMOVE/PREPARE/WAIT/REVIEW actions

Allowed non-Decision-Engine findings:

- `scripts/portfolio/parse_trade_commands.py`, `scripts/portfolio/portfolio_manager.py`, and `scripts/telegram/process_telegram_commands.py` contain BUY/SELL as user transaction-command syntax.
- `scripts/reporting/build_telegram_summary.py` contains action labels only to format existing Decision Engine `final_action` values.
- generated `__pycache__` files may appear in raw grep output and are ignored.

No active upstream runtime layer creates allocation decisions outside `scripts/core/decision_engine.py`.
