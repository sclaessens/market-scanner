# Audit Documentation

This directory contains evidence, reviews, migration records, smoke-test results, cleanup records, and historical execution summaries.

Audit documents are retained for traceability, but they are not the primary source of current project truth. Current normative documentation belongs in `docs/active/`.

## Structure

- `provider_smokes/` — SEC CompanyFacts, real-source, NVDA, persistence and provider smoke evidence
- `legacy_runtime/` — legacy runtime, script-era cleanup, archive readiness and decoupling evidence
- `runtime_boundary/` — runtime boundary migration evidence
- `reset_cleanup/` — reset, cleanup, persistence, provider fixture and architecture review evidence
