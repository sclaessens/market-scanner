## ME02 Goal

ME02 will extract and write the Market Engine functional flow.

ME02 should inspect existing documentation, code, tests, audits, and backlog items that describe:

- user and operator workflows;
- discovery and classification flow;
- source intake flow;
- data availability and failure states;
- portfolio and watchlist interaction boundaries;
- reporting and review boundaries;
- old sprint decisions and deferred product questions.

## Expected Output

ME02 should produce a Market Engine functional flow that is good enough to steer later implementation and tests.

## Anti-Iteration Rule

ME02 must inspect, extract, decide keep / reject / defer, write implications, and move forward.

The functional flow does not need to be theoretically perfect. It must be useful enough to guide implementation and testing.

- Portfolio or watchlist mutation.
- Report generation.
