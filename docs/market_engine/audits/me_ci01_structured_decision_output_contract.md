# ME-CI01 - Structured Decision Output Contract Audit

## Context

ME-RM06 repositioned delivery around this sequence:

```text
Market Engine
-> Structured Artifacts / Structured Decision Output
-> ChatGPT Advisory Layer
-> user
```

It also moved Messenger, Signal, Telegram, email, and later channels into a
compact channel-neutral Notification Layer.

ME-CI01 defines the first contract in that sequence. It creates a stable,
machine-readable Structured Decision Output v1 contract that downstream
consumers can later use without coupling themselves to report text, Telegram
formatting, dashboard layout, or ChatGPT prompt phrasing.

## Decision

Structured Decision Output v1 is approved as a documentation-only contract with:

```text
schema_version: structured-decision-output-v1
artifact_type: market-engine-structured-decision-output
```

The contract defines required top-level objects for:

* instrument identity;
* data coverage;
* decision state;
* scores;
* portfolio context;
* risk;
* price and invalidation levels;
* thesis;
* evidence;
* explainability;
* consumer guidance;
* validation.

The contract is machine-readable first. Human summaries may be generated by
consumers only when `explainability.human_summary_allowed` permits it, and the
summary may not override structured fields.

## Contract scope

ME-CI01 includes:

* contract documentation;
* field semantics;
* required and optional behavior;
* compatibility rules;
* fail-closed rules;
* consumer rules;
* actionable-candidate example artifact;
* blocked/descriptive-only example artifact;
* roadmap and backlog updates.

## Non-goals

ME-CI01 does not:

* implement runtime code;
* change Decision Engine semantics;
* add ChatGPT API integration;
* add Messenger, Signal, Telegram, email, dashboard, or notification
  integration;
* call providers, yfinance, SEC, or EDGAR;
* write portfolio or watchlist state;
* create order, broker, allocation, or execution behavior;
* make human free-form text the source of truth;
* change existing delivery/reporting documents except for roadmap/backlog
  positioning.

## Compatibility rules

Structured Decision Output v1 allows additive backward-compatible evolution.

Compatible changes may add optional fields, reason codes, evidence references,
or consumer labels when they do not weaken existing semantics.

Breaking changes require a new schema version when they rename or remove fields,
change meanings, weaken fail-closed behavior, weaken forbidden-use rules, or add
unapproved action/allocation/order/channel semantics.

## Consumer rules

ChatGPT Advisory Layer may:

* explain structured fields;
* compare artifacts;
* discuss scenarios;
* answer artifact-grounded questions;
* explain blockers and missing evidence.

ChatGPT Advisory Layer must not:

* become the source of truth;
* invent missing values;
* override `blocked`, `descriptive_only`, `is_actionable`, or validation states;
* generate orders, broker instructions, or portfolio mutations.

Notification Layer may:

* consume compact state-change and review-prompt semantics;
* reference ChatGPT for deeper analysis.

Notification Layer must not:

* become the long-form report surface;
* convert candidates into execution instructions;
* hide blockers or missing evidence.

Dashboards and future frontends may:

* display structured state;
* display evidence links;
* display validation and blocker status.

Dashboards and future frontends must not fabricate missing scores, missing
levels, missing portfolio context, or missing actionability.

## Fail-closed behavior

Consumers and future producers must fail closed when required fields are
missing, schema versions are unsupported, artifact type is mismatched, ticker
identity conflicts, numeric levels lack provenance or confidence, validation is
`fail_closed`, coverage is `blocked`, actions are unsupported, or future
versions are supplied without an approved compatibility rule.

Failing closed must produce an explicit blocked or non-consumable state. It must
not produce a default action, allocation, ranking, order, broker instruction, or
notification send.
