# ME-RM07 - Output-First Advisory Roadmap Pivot

Owner roles: Product Owner / Scrum Master / Technical Architect / Development Lead / QA Lead / Governance Auditor

Status: ACTIVE ROADMAP PIVOT

## Objective

Reorient the ChatGPT Advisory Layer roadmap from infrastructure-first iteration to visible product output.

The project has already established the advisory artifact, validation, prompt, grounding, dry-run, hardening, and invocation boundary chain. The next milestone is not another generic boundary or scaffold. The next milestone is a real grounded advisory report generated from a real Market Engine artifact.

## Problem statement

The advisory chain currently proves many internal properties but does not yet provide the user with the concrete result the project is meant to deliver: a readable, grounded stock analysis produced by the application.

The project must therefore stop inserting refinement loops between the completed CI10 boundary and the first real advisory output.

## Product rule

For every next sprint, answer:

```text
What new visible result can the user inspect or use after this sprint that was not available before it?
```

Until the first real grounded advisory report exists, a proposed sprint must be rejected or deferred when its primary deliverable is only:

- a contract;
- a validator;
- a taxonomy;
- a scaffold;
- a boundary;
- generic hardening;
- generic provider abstraction;
- multi-provider routing;
- fallback-model policy;
- delivery plumbing without usable advisory content.

Exception: a narrow corrective sprint is allowed only when an actual end-to-end run exposes a concrete blocker that prevents the visible output milestone.

## Output-first sequence

```text
ME-CI11 - Generate first real grounded advisory output from a Market Engine artifact
  -> ME-CI12 - Render grounded response as a high-quality human-readable stock report
  -> ME-CI13 - Run real advisory reports for NVDA, AMD, and ASML
  -> ME-CI14 - Generate portfolio-aware advisory output from approved portfolio context
  -> ME-CI15 - Expose advisory output through the application interface
```

## ME-CI11 - First real grounded advisory output

Status: NEXT ACTIVE ADVISORY SPRINT

Goal: produce one real readable advisory report for one real ticker from an existing Market Engine artifact.

Required execution path:

```text
real Market Engine ticker artifact
  -> CI06 validation
  -> CI08/CI09 prompt package path
  -> controlled model invocation under CI10 boundary
  -> raw response capture
  -> strict structured parse
  -> mandatory CI09 grounding validation
  -> local human-readable advisory report
```

Hard Definition of Done:

- at least one real ticker is used;
- a real model invocation occurs under the approved non-production boundary;
- raw response is captured;
- structured response is parsed strictly;
- CI09 grounding validation runs;
- an ungrounded or blocked result fails closed;
- a grounded result produces a readable `advisory_report.md`;
- the report is traceable through source artifact, prompt package, invocation metadata, raw response, parsed response, grounding result, and manifest;
- no successful report is produced from failed, malformed, ungrounded, or blocked output.

Recommended artifact set:

```text
artifacts/market_engine/<ci11-run>/<ticker>/
  advisory_artifact.json
  prompt_package.json
  invocation_request.json
  raw_model_response.json
  parsed_response.json
  grounding_result.json
  advisory_report.md
  manifest.json
```

Governance boundary:

- non-production;
- one controlled provider;
- one controlled model profile;
- non-streaming;
- no model tools;
- no model-side browsing;
- no broker action;
- no portfolio mutation;
- no watchlist mutation;
- no allocation authority;
- no sizing authority;
- no execution authority;
- no delivery adapter.

## ME-CI12 - Human-readable stock report

Goal: improve the grounded response into a consistently readable stock report format based on actual CI11 run evidence.

Visible result:

```text
A user can open a report and understand:
- current state;
- supporting evidence;
- opposing evidence;
- blockers;
- uncertainty;
- freshness;
- current recommendation state when upstream-supported;
- what would improve the case;
- what would weaken the case;
- what cannot be determined.
```

ME-CI12 may improve rendering and report structure only after CI11 proves the real generation path. It must not become another pre-output contract sprint.

## ME-CI13 - Real ticker bundle

Goal: run the complete advisory-output path for:

```text
NVDA
AMD
ASML
```

Visible result: three comparable grounded advisory reports generated by the application.

The sprint must expose real differences in source readiness, coverage, blockers, freshness, and advisory eligibility. It must not hide weaker coverage to force uniform outputs.

## ME-CI14 - Portfolio-aware advisory output

Goal: use approved Portfolio Intelligence context to generate grounded answers that distinguish standalone stock assessment from portfolio fit.

Visible result: a readable portfolio-aware report or answer that can explain, when supported:

- whether the user already holds the ticker;
- concentration context;
- overlap and exposure context;
- portfolio relevance;
- position-management explanation from approved upstream context.

It must not invent sizing, target weights, allocation, or trade execution authority.

## ME-CI15 - Application exposure

Goal: expose generated advisory output through the application interface.

Visible result: the user can select or request a ticker analysis and view the grounded advisory output without manually browsing artifact directories.

The interface must preserve grounding state, disclosures, blockers, uncertainty, freshness, and failure states.

## Deferred until after visible output

The following are explicitly deferred until actual output evidence proves they are needed:

- multi-provider routing;
- fallback model policy;
- provider abstraction beyond the first controlled path;
- additional generic grounding hardening;
- generic parser framework expansion;
- external browsing contract;
- notification adapters;
- broad delivery orchestration;
- advanced observability beyond what the first real run requires.

## Governance decision

The first real advisory report is now the primary advisory milestone.

Roadmap changes after this pivot must preserve the rule:

```text
actual run evidence first
narrow corrective infrastructure second
visible user output remains the milestone
```

No new refinement sprint may be inserted before ME-CI11 merely because additional theoretical edge cases can be imagined.
