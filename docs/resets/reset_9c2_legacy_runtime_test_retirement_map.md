# RESET-9C2: Legacy Runtime Test Retirement Map

## 1. Purpose

RESET-9C2 inventories and classifies tests that still protect, reference, or guard legacy runtime behavior.

This document is a retirement map only. It does not authorize deleting, moving, archiving, renaming, or refactoring any test.

## 2. Scope

In scope:

- test files under `tests/`;
- tests that import or reference `scripts`;
- tests that reference legacy data or report paths;
- tests that encode provider, SEC, Telegram, reporting, portfolio, Decision Engine, validation, fundamentals, diagnostics, or orchestration behavior;
- v2 tests that already protect `src/market_scanner` and v2 fixture contracts.

Out of scope:

- test cleanup execution;
- runtime cleanup execution;
- v2 implementation;
- archive/delete execution;
- provider or network execution;
- production pipeline execution.

## 11. Knowledge Translation Requirements

Translate or explicitly reject the following before any legacy test retirement:

- validation schema and entry-quality expectations;
- context row preservation, sector-relative enrichment, and nullable metadata rules;
- fundamentals history schema, metrics formulas, quality states, analysis states, missing-value behavior, and stale-source handling;
- timing state metadata and deterministic ordering;
- portfolio transaction, position, metadata, exposure, and source-boundary rules;
- Decision Engine final-action/allocation/execution/arbitration authority and source provenance;
- reporting row preservation, traceability, log metadata, deterministic communication output, and Telegram artifact generation;
- SEC local transformation, ticker/CIK mapping, and no-network guardrails;
- diagnostics and historical evidence capture if still required;
- orchestration/operator visibility and manual fallback expectations.

## 12. Tests That Must Not Be Retired Yet

Do not retire:

- any test that protects `scripts/run_scan.py` or `scripts/run_full_pipeline.py`;
- any test that protects legacy Decision Engine authority;
- any test that protects provider, SEC, Telegram, or portfolio command boundaries;
- any test that verifies no hidden filtering, no upstream tradeability, row preservation, or missing-not-zero semantics;
- any v2 active or transition guardrail test;
- any test whose protected behavior has not been translated into v2 doctrine/contracts/tests.

## 14. Stop Conditions

Any future test retirement work must stop if:

- a test imports `scripts` and no v2 replacement test exists;
- a test protects manual fallback behavior;
- a test protects Decision Engine authority;
- a test references provider, SEC, Telegram, portfolio, generated data, or report behavior that has not been reapproved;
- a proposed change would touch `tests/`, `scripts/`, `src/`, `data/`, `reports/`, or workflows outside an explicit implementation scope;
- a proposed cleanup would require running the production pipeline, Telegram scripts, SEC diagnostics, provider calls, or network calls.

## 15. Decision

Decision: `DO_NOT_DELETE_OR_ARCHIVE_LEGACY_TESTS_YET`

Rationale:

- legacy runtime remains the only complete manual fallback;
- many tests encode domain and edge-case knowledge required for v2 translation;
- v2 replacements are incomplete or scaffold-only for most runtime layers;
- SEC, provider, Telegram, portfolio, reporting, and generated-data tests require reapproval or policy decisions;
- tests are needed to make knowledge extraction safe.
